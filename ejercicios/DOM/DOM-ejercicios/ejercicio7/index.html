Ejercicio 7 - La familia del DOM. Padres e hijos

La siguiente lista tendra un padre y este tendrá unos hijos

  • Yo soy el primer hijo de la lista
  • Yo soy un hijo intermedio de la lista
  • Yo soy el ultimo hijo de la lista

Debajo de este parrafo hay cuatro parrafos contenidos dentro del mismo "div"

Yo soy el primer hijo del div

Yo soy uno de los hijos intermedios del div

Yo tambien soy uno de los hijos intermedios del div

Yo soy el ultimo hijo del div

Aqui se mostrara el resultado de pulsar los botones

Para movernos por el arbol DOM y trabajar con sus elementos hay distintas funciones que podemos usar


Todo elemento que este en el DOM tiene un elemento padre que lo contiene hasta llegar al elemento html que es padre de padres.


Cada elemento del DOM es un nodo y hay tres tipos de nodos: elemento, texto y comentarios. Sabiendo esto las propiedades suelen dividirse en aquellas que contienen la palabra Element y las que no, la diferencia radica en si capturan nodos de tipo elemento o de cualquier tipo


Para acceder al padre de un elemento podemos hacerlo con parentNode y con parentElement la diferencia esta principalmente en el tipo de objeto que devuelven y en el caso de no encontrar al padre. En la consola podemos ver que devuelven todo el html que contienen en ambos casos


Para acceder a los hijos de un elemento usaremos childNodes que nos devolvera un objeto de tipo NodeList con los hijos, este elemento sera iterable. Tambien podremos usar children solo que en este caso solo capturara elementos y devolvera un HTMLCollection. El primero se suele usar mas para manipualcion de los elementos mientas que el segundo para navegar por ellos. En la consola podemos ver la diferencia de tipos. Tambien podemos ver la diferencia en el resultado ya que muestra las tabulaciones que incluye la lista al haber sido capturada con childNodes


Para acceder al primer hijo podemos usar firstChild o firstElementChild. Es muy importante tener en cuenta las indexaciones de codigo ya que este tambien añade, al usar esta propiedad, espacios vacios. Por eso aparece como undefined el contenido de los nodos capturados con el primer modo


Para acceder al ultimo hijo podemos usar lastChild o lastElementChild. Funciona igual que firstChild y firstElementChild
