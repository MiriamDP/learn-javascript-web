Ejercicio - Guardando en localStorage




Podemos guardar de dos formas en localStorage. Imaginemos que tenenos una tabla con los usuarios de nuestra aplicacion. Los datos de esta tabla podriamos almacenarlos por filas o podriamos guardar la tabla completa. En este primer caso guardaremos cada fila de la tabla como una entrada en el localStorage


Para guardar un elemento localStorage.addItem (clave, valores). Valores sera un objeto JSON por lo que habra que convertirlo a string para poder almacenarlo mediante JSON.stringify(valores) .Podemos ver en las herramientas de desarrollador en la pestaña Aplicacion que se ha creado una entrada en el localStorage con el usuario añadido. En key estara el nombre de usuario y en value los valores de ese


localStorage es un array escalar. Podremos recorrelo de diversas formas. Forma 1con un bucle for. Donde cada entrada sera un array [key, value], para acceder a las keys usaremos localStorage.key() y para los valores tendremos que acceder al item con la key dada y pasarlo a formato JSON de esa forma podremos acceder a propiedades concretas del objeto


Forma 2 Con un bucle for of de las claves. Haciendo un Objects.keys (localStorage) que nos devovlera un array con todas las claves y que recorreremos con el bucle for of


Forma 3 Con un bucle forearch al transformar el localStorage en un array de arrays Objecs.entries (localStorage).foreach()


Forma 4 Con un bucle for of y Object.entries. Y forma 5 con un bucle for in del propio localStorage aunque no es recomendable


Para eliminar un elemento bastara con usar el metodo removeItem(clave)


Esta forma de guardar en el localStorage es bastante inconveniente en especial si tenemos muchos datos o esos datos proceden de tablas distintas, en ese caso tendriamos informacion mezclada


Tambien al igual que ocurria con cookies no todos los navegadores permiten acceder al localStorage. En este caso se recomienda trabajar con Chrome en lugar de con Firefox
